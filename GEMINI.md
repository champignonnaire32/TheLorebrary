# Project Panic Catchup (The Distiller Platform)

## 🎯 The Core Mission
This project is a fast, SEO-optimized static website built to host "High-Density Narrative Summaries" of long book series (e.g., Sun Eater, Stormlight Archive). The target audience is returning readers who already own the books but need a "concrete refresher" before a new sequel release.

**Key Mandates:**
*   **Concrete Narrative:** Plot points and actions only. No thematic analysis or flowery prose.
*   **Strict Length:** 10-15 sentences per chapter chunk.
*   **Minimalist UX:** An "Archive-like" aesthetic (dark/light mode, clean typography, fast loading).
*   **SEO Strategy:** Targeting "Panic Recaps" (e.g., "Empire of Silence recap chapters 36-40").

## 🏗️ Architecture & Tech Stack
We are using a modern Static Site Generation (SSG) approach.
*   **Framework:** Astro (v5) - Chosen for its native Markdown ingestion (Content Collections) and zero-JS output by default.
*   **Styling:** Tailwind CSS (v4) - Using the `@tailwindcss/typography` plugin to automatically format the AI-generated Markdown into beautiful, legible prose.
*   **Interactivity:** React - Used exclusively for "Islands" of interactivity, specifically the `ReaderControls` floating menu.
*   **Data Pipeline:** Python scripts (`smart_pdf_splitter.py`, `distiller_auto.py`) process PDFs into `.md` chunks, which are then split and ingested into Astro.

## 📝 Directory Structure & Data Flow
1.  **Raw Data:** `/src/data/raw-summaries/` holds the giant `_Summary.md` files generated by the Python pipeline.
2.  **Ingestion:** The Node script `scripts/split-summaries.js` reads those raw files, splits them by chapter (`###`), cleans the AI metadata, and outputs atomic `chapter-X.md` files with YAML frontmatter.
3.  **Content:** `/src/content/summaries/` holds the final, atomic chapter files that Astro type-checks and renders.
4.  **Routing:** 
    *   `/` -> `index.astro` (Lists all available Books).
    *   `/[series]/[book]/` -> `index.astro` (Lists all 5-chapter chunks for that Book).
    *   `/[series]/[book]/[range]` -> `[range].astro` (The actual reading page, displaying 5 chapters at a time).
