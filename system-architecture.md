# The Lorebrary - System Architecture

This document details the frontend technology stack, structural design, and data flow of the application. For backend Python parsing logic, see `the-distiller-engineering.md`.

## 🏗️ Architecture & Tech Stack
We are using a modern Static Site Generation (SSG) approach to ensure lightning-fast, zero-JS page loads.

*   **Framework:** Astro (v5) - Chosen for its native Markdown ingestion (Content Collections) and zero-JS output by default.
*   **Styling:** Tailwind CSS (v4) - Using the `@tailwindcss/typography` plugin to automatically format the AI-generated Markdown into beautiful, legible prose.
*   **Interactivity:** React - Used exclusively for "Islands" of interactivity (`ReaderControls.tsx`, `SearchBar.tsx`, `ContinueReading.tsx`) that require client-side state.

## 🎨 Frontend Design System Implementation
A full visual identity overhaul was applied on top of the SSG architecture using these technical implementations:

*   **Fonts:** Google Fonts — `Cinzel` (display/headers, classical codex feel) + `Lora` (body/reading, premium book readability). Loaded via preconnect in `Layout.astro`.
*   **Color palette:** Custom `@theme` tokens in Tailwind v4 — `ink-*` (deep navy-black for dark mode), `cream-*` (warm parchment for light mode), `gold-*` (amber accent replacing all blue). Defined in `src/styles/global.css`.
*   **Prose:** `@tailwindcss/typography` overrides for Cinzel headings in gold, Lora body text, 1.95 line-height, decorative gradient HR rule.
*   **Interactive components:** `ReaderControls.tsx` FAB and `SearchBar.tsx` updated to match the ink/gold palette using hardcoded hex values (preserves existing `px`-based sizing that prevents layout scaling issues).
*   **Sepia mode:** Third theme option (warm sepia) in `ReaderControls.tsx`. Implemented via CSS custom property overrides on `html.sepia` — no HTML changes needed. Cascades through all Tailwind utilities automatically.

## 📝 Directory Structure & Data Flow

1.  **Raw Data:** `/src/data/raw-summaries/` holds the giant `_Summary.md` files generated by the Python pipeline.
2.  **Ingestion:** The Node script `scripts/split-summaries.js` reads the pre-chunked `*_Summary.md` files, wraps each one in Astro-compatible YAML frontmatter (series name, book name, bookOrder, range, startChapter), and writes them to the content collection.
3.  **Content:** `/src/content/summaries/` holds the final, atomic chapter files that Astro type-checks and renders.
4.  **Series Metadata:** `src/data/series-metadata.ts` — **manually maintained** file with author name, genre, and 5-sentence blurb per series.

## 🛒 Purchase Buttons

Every book index page (`/[series]/[book]/`) displays a "Get the Book" section with two purchase links and a library note. These are **fully automatic** — no manual work is needed when adding a new book or series.

**Implementation file:** `src/pages/[series]/[book]/index.astro`

**URL construction** (in the frontmatter script block):
```typescript
const q = encodeURIComponent(bookName);
const bookshopUrl = `https://bookshop.org/search?keywords=${q}`;
const bnUrl       = `https://www.barnesandnoble.com/s/${q}`;
```
Both URLs use the book's title as a search query, which reliably surfaces the correct book as the first result for all series currently in the archive.

**Visual design:**
- Both buttons use a solid gold fill (`bg-gold-500 dark:bg-gold-400`) with ink text (`text-ink-950`) — consistent with the primary CTA treatment used elsewhere on the site.
- Bookshop.org is listed first and includes a `"supports indie bookshops"` caption in muted text to subtly signal it as the preferred option without explicit editorializing.
- Both buttons are `flex-1` inside a `max-w-md mx-auto` container, ensuring equal width regardless of text length.
- Below the buttons, a small italic line reads: *"Haven't grabbed a copy yet? Also available at your local library or on [Libby](https://libbyapp.com)."* The phrasing is deliberately neutral — it notes library availability without framing it as an alternative to purchasing, which is important for maintaining good faith with authors and publishers.

**Amazon is intentionally excluded.** The decision was made to omit Amazon links entirely.

**Future upgrade (ISBN-based deep links):** See `roadmap-and-design.md` for the planned upgrade path to store ISBN-13s per book in `series-metadata.ts` and construct direct product page URLs instead of search results.

## 🛣️ Routing Architecture

The application relies on file-based routing and dynamic parameters:
*   `/` → `index.astro` (Landing page with legal disclaimers).
*   `/archive` → `archive.astro` (Series grid with Continue Reading strip + search).
*   `/[series]/` → `index.astro` (Series page with blurb + book list).
*   `/[series]/[book]/` → `index.astro` (Chunk grid with read-state checkmarks + progress bar).
*   `/[series]/[book]/[range]` → `[range].astro` (Reading page — 5 chapters at a time).
*   `/[series]/[book]/full` → `full.astro` (Mega-Catchup — entire book as one continuous scroll).
